# BJTU 智慧食堂仿真系统

北京交通大学智慧食堂实时监测仿真系统，使用 Python Flask + SQLite + 原生 JavaScript 实现，提供窗口排队、餐桌分布、统计分析等功能。

## 项目简介

本系统是一个智慧食堂仿真平台，模拟食堂运营中的各项数据指标，帮助管理者直观了解食堂运营状况。参与开发人员：软件综合实训S002小组，成员为雷佐、李佳勋、杨添博。

### 核心功能

- **实时数据监测**: 窗口排队人数、餐桌占用状态、等待时间估算
- **仿真参数配置**: 支持自定义排队人数范围、窗口关闭概率、餐桌占用概率等参数
- **场景预设**: 提供平峰、午餐高峰、晚高峰、极端拥堵、设备维护、空闲时段等预设场景
- **数据可视化**: 实时趋势图表展示人数、等待时间、占用率变化
- **窗口管理**: 支持新增和删除食堂窗口

## 快速开始

### 环境要求

- Python 3.8+
- pip 包管理器
- Docker（可选，用于容器化部署）

### 开发模式运行

1. **克隆项目**
```bash
cd "c:\Users\15060\Desktop\软件综合 实训\CSS"
```

2. **激活虚拟环境**
```bash
.venv\Scripts\activate  # Windows
# 或
source .venv/bin/activate  # Linux/Mac
```

3. **安装依赖**
```bash
pip install -r css/requirements.txt
```

4. **初始化数据库**
```bash
cd css
python init_database.py
```

5. **启动服务**
```bash
python server.py
```

6. **访问系统**
打开浏览器访问: `http://localhost:8888`

### Docker 容器化部署

```bash
cd css
docker-compose up -d
```

服务将在 `http://localhost:3000` 运行。

## API 接口

### 仿真参数管理

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/settings` | 获取当前仿真参数 |
| PUT | `/api/settings` | 更新仿真参数 |

**参数说明**:

| 参数 | 类型 | 范围 | 默认值 | 说明 |
|------|------|------|--------|------|
| queueMin | int | 0-50 | 1 | 最少排队人数 |
| queueMax | int | 0-80 | 15 | 最多排队人数 |
| closedRate | int | 0-100 | 10 | 窗口关闭概率(%) |
| occupancyRate | int | 0-100 | 60 | 餐桌占用概率(%) |
| maintenanceRate | int | 0-100 | 2 | 维护概率(%) |
| tableCount | int | 10-160 | 80 | 餐桌数量 |
| refreshInterval | int | 1-15 | 3 | 刷新间隔(秒) |

### 窗口管理

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/windows` | 获取所有窗口实时数据 |
| POST | `/api/windows` | 新增窗口 |
| DELETE | `/api/windows/<id>` | 删除指定窗口 |

**POST 请求示例**:
```json
{
  "name": "特色窗口",
  "category": "小吃"
}
```

### 餐桌管理

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/tables` | 获取所有餐桌实时数据 |

### 统计数据

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/statistics` | 获取食堂统计数据 |

### 菜单数据

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/menu` | 获取今日菜单数据 |

## 场景预设

系统提供以下预设场景，可一键切换：

| 场景   | 特点           | 适用时段                     |
| ---- | ------------ | ------------------------ |
| 平峰   | 排队人数少，占用率低   | 10:00-11:00, 14:00-16:00 |
| 午餐高峰 | 排队人数多，占用率高   | 12:00-12:30              |
| 晚高峰  | 排队适中，占用率较高   | 18:00-18:30              |
| 极端拥堵 | 排队人数极高，接近满负荷 | 突发状况模拟                   |
| 设备维护 | 大量窗口/餐桌维护中   | 设备检修时段                   |
| 空闲时段 | 人数稀少，多数窗口关闭  | 非用餐时间                    |
|      |              |                          |

## 数据库结构

### windows 表（窗口信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | TEXT | 窗口名称 |
| category | TEXT | 窗口分类 |
| queue_count | INTEGER | 排队人数 |
| wait_time | TEXT | 预计等待时间 |
| status | TEXT | 状态(open/closed) |
| last_update | TEXT | 最后更新时间 |

### tables 表（餐桌信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| is_occupied | INTEGER | 是否占用(0/1) |
| capacity | INTEGER | 容纳人数 |
| current_people | INTEGER | 当前人数 |
| last_update | TEXT | 最后更新时间 |

### menu 表（菜单信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | TEXT | 菜品名称 |
| price | INTEGER | 价格(元) |
| rating | REAL | 评分(0-5) |
| sold | INTEGER | 销量 |

## 功能特性

1. **实时数据更新**: 自动定时刷新，默认每 3 秒更新一次
2. **数据可视化**: 使用 Canvas 绘制趋势图表，直观展示数据变化
3. **参数限制**: 所有参数均有合理范围限制，防止参数失控
4. **响应式设计**: 适配桌面端和移动端显示
5. **状态指示**: 通过颜色和图标直观区分空闲/占用/维护状态

## 使用说明

1. **查看统计面板**: 首页顶部显示食堂整体运营数据
2. **调整仿真参数**: 使用滑块调整各项参数，点击"应用参数"保存
3. **选择场景预设**: 点击场景按钮快速切换到预设场景
4. **管理窗口**: 在窗口排队区域新增或删除窗口
5. **查看餐桌分布**: 底部餐桌分布图实时显示各餐桌状态
6. **查看趋势图表**: 统计区域展示历史数据趋势
